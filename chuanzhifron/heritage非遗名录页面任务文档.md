# heritage非遗名录页面任务文档

## 1. 概述

本文档旨在为团队成员分配heritage非遗名录页面开发相关的具体任务，确保团队协作高效有序。文档将详细说明各项任务的具体内容、操作方法以及任务对最终产品的影响。

## 2. 图片素材收集任务

### 2.1 非遗项目图片收集
**任务描述：** 收集至少30个代表性非遗项目的高清图片

**具体内容：**
1. 传统技艺类（10个）：景泰蓝、苏绣、蜀锦、景德镇瓷器、紫砂陶、剪纸、木版年画、漆器、竹编、蜡染
2. 传统美术类（5个）：国画、书法、篆刻、雕塑、唐卡
3. 传统音乐类（3个）：古琴、琵琶、二胡
4. 传统舞蹈类（2个）：孔雀舞、秧歌
5. 传统戏剧类（5个）：京剧、昆曲、越剧、黄梅戏、川剧
6. 曲艺类（3个）：相声、评书、快板
7. 民俗类（2个）：春节、端午节

**操作指南：**
1. 通过搜索引擎、非遗保护官网、文化机构网站等渠道寻找合适的高清图片
2. 确保图片分辨率不低于800px宽度
3. 注意版权问题，优先选用免费可商用或已获得授权的图片
4. 将图片统一命名为：heritage_1.jpg, heritage_2.jpg ... heritage_30.jpg

**为什么要做这项工作：**
非遗项目图片是用户了解非遗文化最直观的方式，高质量的图片能够充分展现非遗技艺的精美和独特魅力，提升用户浏览体验。

**对应界面效果：**
在非遗名录页面以瀑布流形式展示，每张图片配有项目名称、级别标签和地区信息。

### 2.2 分类图标收集
**任务描述：** 收集7个非遗分类的图标图片

**具体内容：**
1. 传统技艺分类图标1个
2. 传统美术分类图标1个
3. 传统音乐分类图标1个
4. 传统舞蹈分类图标1个
5. 传统戏剧分类图标1个
6. 曲艺分类图标1个
7. 民俗分类图标1个

**操作指南：**
1. 寻找简洁明了、易于识别的矢量图标
2. 图标风格保持统一，建议使用线性图标
3. 尺寸为80px*80px，PNG格式，支持透明背景
4. 命名规范：category_skill.png, category_art.png等

**为什么要做这项工作：**
分类图标帮助用户快速理解和区分不同的非遗门类，提升筛选效率和用户体验。

**对应界面效果：**
在页面顶部筛选栏中水平排列展示，选中状态会有高亮显示。

## 3. 文案内容撰写任务

### 3.1 非遗项目基本信息撰写
**任务描述：** 为每个非遗项目撰写基本信息

**具体内容：**
1. 为30个非遗项目分别撰写准确的项目名称
2. 确定每个项目的保护级别（国家级、省级、市级）
3. 确定每个项目的所属地区
4. 为每个项目撰写一段100字以内的简介

**操作指南：**
1. 项目名称要准确权威，可参考官方表述
2. 保护级别要核实准确
3. 所属地区精确到省市级别
4. 简介要突出该项目的核心特色和文化价值，语言要通俗易懂

**为什么要做这项工作：**
准确的基本信息是用户了解非遗项目的基础，有助于用户快速掌握项目要点。

**对应界面效果：**
在非遗项目卡片上显示项目名称、级别标签、地区和简介。

### 3.2 非遗项目详细介绍撰写
**任务描述：** 为每个非遗项目撰写详细介绍

**具体内容：**
1. 为30个非遗项目分别撰写500字以上的详细介绍
2. 撰写每个项目的历史渊源（200字以上）
3. 撰写每个项目的传承现状（200字以上）
4. 为部分重点项目撰写制作工艺流程（可选）

**操作指南：**
1. 详细介绍要全面涵盖项目的历史、特点、价值等方面
2. 历史渊源要追溯项目起源和发展脉络
3. 传承现状要客观描述当前保护和传承情况
4. 语言要生动有趣，避免过于学术化

**为什么要做这项工作：**
详细内容是用户深入了解非遗项目的重要途径，丰富的信息能提升用户的学习和探索体验。

**对应界面效果：**
在项目详情页面展示，用户点击项目卡片后可查看详细内容。

### 3.3 分类名称文案撰写
**任务描述：** 确定各分类的准确名称

**具体内容：**
1. 确定7个非遗分类的准确中文名称

**操作指南：**
1. 分类名称要准确反映该类别非遗的特点
2. 名称要符合国家非遗分类标准
3. 保持命名一致性

**为什么要做这项工作：**
准确的分类名称是用户理解和筛选非遗内容的重要标识。

**对应界面效果：**
在筛选栏中显示分类名称，用户可通过点击进行筛选。

## 4. 数据整理与格式化任务

### 4.1 非遗项目数据整理
**任务描述：** 将收集到的非遗项目信息整理成结构化数据

**具体内容：**
1. 建立非遗项目信息表格，包含字段：ID、名称、分类、级别、地区、简介、详细介绍、历史渊源、传承现状
2. 确保每个字段信息完整准确
3. 按分类和级别进行排序

**操作指南：**
1. 使用Excel或Google Sheets建立信息表格
2. 每个项目一行，字段列为不同属性
3. 定期备份数据，避免丢失

**为什么要做这项工作：**
结构化数据便于技术人员开发使用，也能确保信息的一致性和完整性。

**对应界面效果：**
整理好的数据将成为页面展示的基础，直接影响用户看到的内容。

### 4.2 图片与数据关联
**任务描述：** 建立图片与非遗项目数据的对应关系

**具体内容：**
1. 为每张非遗项目图片标注对应的项目ID
2. 建立图片名称与项目ID的对照表
3. 确认每张图片的版权信息

**操作指南：**
1. 建立图片清单表格，包含图片名称、对应项目、版权信息等字段
2. 核对图片与项目信息的匹配准确性
3. 整理图片版权相关信息

**为什么要做这项工作：**
确保每张图片都能正确显示在对应的非遗项目中，避免错位或缺失。

**对应界面效果：**
用户在浏览非遗项目时能看到正确的配图。

## 5. 交付要求

### 5.1 交付内容
1. 所有图片素材按分类放入对应文件夹
2. 非遗项目信息数据表格
3. 图片与项目对照表
4. 所有文案内容整理成文档

### 5.2 交付时间
建议在5个工作日内完成所有素材收集和文案撰写工作

### 5.3 质量要求
1. 图片清晰度达到展示要求
2. 文案内容准确无误，无明显语法错误
3. 命名规范统一
4. 版权问题已妥善处理
5. 数据完整性和一致性良好
